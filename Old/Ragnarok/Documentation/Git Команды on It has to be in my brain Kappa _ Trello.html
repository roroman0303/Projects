    git status - показывает изменённые файлы и файлы подготовленные к коммиту
    git add - добавляет файлы в подготовленные для коммита, для удаления файлов из подготовленных - git reset
    git push - заливает все изменения-коммиты на удаленный сервер
    git checkout <branch> <filename> - берет файлы из другой ветки branch и подготавливает их к коммиту (пример - git checkout pts2 ./zone/idtable.txt)
    git checkout <branch> - меняет локальную ветку
    git checkout . - откатить все изменения
    git checkout <filename> - откатить изменения одного файла
    git reset <filename> - убирает файл из подготовленных (обратная команда git add)
    git pull --rebase - для того, чтобы поднять коммит в списке коммитов, если кто-то в процессе изменений тоже что-то изменил и запушил (не обязательно, но очень желательно)
    git cherry-pick <other_branch> - для того, чтобы забрать коммиты из другой ветки в свою ветку
    git revert -n pts3~7..pts3 - откатить 7 последних коммитов (-n чтобы не создавать коммит)
    git revert -n [commit_like_239f9382f] - откатить коммит (-n чтобы не создавать коммит)
    git cherry-pick -n pts3~7..pts3
    (Другие полезные https://wiki.inn.ru/display/RO/%5BRO%5D+Gitlab)

    Обычно работа с Git выглядит так:
    git pull
    меняем нужные файлы
    git status (должно быть modified красным - т.е. пока не подготовлено)
    git add .
    git status (должно быть modified зеленым - подготовлено)
    git commit -m "сообщение поясняющее коммит"
    git push
    (если git push не прошел, значит кто-то уже изменил с момента пула, тогда:
    git pull
    открывается блокнот, можно ничего не менять
    git rebase для поднятия своих коммитов
    git push)
    Затем проверить в чате ro-git изменения.

    Инфа для любителей горяченького (merge конфликты)
    Скачиваем kdiff3 https://sourceforge.net/projects/kdiff3/files/latest/download
    После этого делаем его дефолтным инструментом для разрешения конфликтов гита
    git config --global merge.tool kdiff3
    git config --global mergetool.kdiff3.path "C:/Program Files/KDiff3/kdiff3.exe"
    git config --global mergetool.kdiff3.trustExitCode false
    git config --global diff.guitool kdiff3
    git config --global difftool.kdiff3.path "C:/Program Files/KDiff3/kdiff3.exe"
    git config --global difftool.kdiff3.trustExitCode false
    после этого, если у вас возникает конфликт, пишем в гит консоли: git mergetool
    иногда он решает их автоматом, если не получается - откроется kdiff3
